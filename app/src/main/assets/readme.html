一些说明

想使用该App有什么要求？

你必须拥有Xposed框架，并激活该模块，然后重启。假如重启后仍无法生效，或者以前能用更新后突然不能了，那多重启几下（或卸载后重装）。
如果你装上之后就是无效，那抱歉我也没有办法。假如你没有Xposed框架，也希望使用这样的功能，推荐酷友的一款类似应用：用默认浏览器打开链接-酷安网

该App能支持哪些应用？

由两部分组成：一种是我自己维护的（用户量比较大，使用场景比较多的），一种是用户自定义的（并不是太普及的应用）。
我打算维护的App为：QQ各系列、微信、微博、贴吧。但是比如知乎这些App其实也是支持的，但考虑到并不是人人需要，就得有需求的用户自行添加了。

屏蔽的应用里有些页面没法用外置浏览器打开？

一部分是我遗漏了，另一部分是存在一些问题。请浏览完本页面的“当前问题”部分，如果没有解释，请把详细的比如哪个页面、什么样的链接，最好有截图。这些信息评论告诉我。（目前会固定看Google Play与酷安的评论）

屏蔽的应用里有些不该用外置浏览器打开的页面被迫打开了？

假如是自带的几个App，应该是我遗漏了，请你反馈给我。同时你也可以通过“添加白名单”功能解决，详情请看“使用方法”部分

使用方法

如何自定义屏蔽应用？

一共分成三步：首先点“添加应用”，输入应用的包名，随后重启手机；重启后打开本App，把提示里的链接放入要屏蔽的App中（比如聊天、发表动态），然后打开它；如果成功，则会自动出现“步骤三”的页面，重启手机即可。

应用的包名是什么？为什么我不可以选择应用？

我相信有能力使用Xposed的用户应该能通过搜索解决包名是什么这个问题。至于为什么没做自动选择应用功能，因为这写起来比较烦，我还是希望把精力放在核心代码上。

步骤二里我打开链接了，可是不会自动出来“步骤三”的页面啊？/我步骤都完成了，但是还是无效啊？

有些是App做了链接跳转（比如微博的所有链接都会变成t.cn的链接），有些是App代码实在是写的比较奇葩，请把具体情况评论反馈给我。在反馈之前我希望你能确认下不是自己的问题，比如没重启或者找不到输链接的地方什么的（比如知乎的手机端你输链接，是不会被自动识别成链接的，需要到网页版编辑答案。这种请自己想办法）。我收到反馈之后会自己试试的，假如兼容不了我也会回复你的。

是不是在完成步骤后一定得重启手机？

其实不重启也是可以的，但是必须得杀掉屏蔽应用的后台。考虑到国产应用们的流氓，所以我还是说重启，防止杀不干净。

这个页面我可以用外置浏览器打开了，但是另一个页面却不行？

实现比较规范的应用比如QQ是只有一个内置浏览器页面的，但是比如贴吧微博这种我就不忍心说了，请尝试长按需要修改的App，选择“添加规则”。这种情况也希望你能反馈给我，因为我并不使用贴吧微博，有些页面我没发现。

“添加规则”是干什么的，如何使用？

“添加规则”就是解决上面这个问题的，一些应用可能有好几个“内置浏览器”，我可能会有疏忽，你可以尝试自己添加。使用步骤与屏蔽应用相同。

有些不该被外置浏览器打开的页面被打开了，怎么办？/如何使用“添加白名单”功能？

这是因为有些App的某些页面其实还是用网页实现的，比如QQ的空间模块里，留言部分跟情侣空间两个按钮假如点击，会在本App的1.1版本用外置浏览器打开。我采用了一种简单粗暴的方法：这些链接都是应用的页面（非用户发布），所以直接忽略对应的域名就好了，比如QQ我自带忽略了h5.qzone.qq.com、微博我忽略了card.weibo.com。
使用方法：长按对应App，选择“添加白名单”，添加即可。具体添加的内容为你误打开页面的域名。

QQ的群公告里的链接没法用外置浏览器打开？

这个没法解决，因为整个群公告部分都是处在内置浏览器之中，因此是没法获取到某条群公告里的某个链接的。

微博打开B站的链接还是内置浏览器？

因为B站的分享都采用了share.acg.tv的链接，而这个链接又不会被B站App解析，所以还是得先打开外置浏览器，再打开B站App，因此还不如用内置浏览器得了。
总而言之，是B站的锅，不怪我。
当然，其他一些用了微博分享SDK的应用也是如此，想区分就是看在它们的链接，使用了SDK的是有名字的，而直接的链接是只有“网页链接”四个字。这种要想处理，意外可能很多，所以我放弃兼容这块。

有些链接开启了屏蔽，使用外置浏览器打开了，却访问不了？

因为有的App太不要脸了！代码还没写规范！没错我说的就是微博跟贴吧！！！它们把统计信息作为链接的参数放进去，但是却因为加了参数导致某些网站处理不了。而正常的应用比如腾讯是在Intent里传参的。

微信为什么只能一部分链接用外置浏览器打开？

微信的公众号页面，还有一些需要用微信登陆的页面，都是必须使用内置浏览器打开的。因此为了避免误杀，只有纯链接才会用外置浏览器打开。
所以比如像B站这样用了微信分享SDK的分享链接（一个有标题有简介有预览图，下方标注了”哔哩哔哩动画“这样的），我只能不管它。原因跟微博一样，只能怪这些大厂应用那么喜欢自己搞一套分享SDK了。
如果这种处理还有误杀，请停用该规则。

为什么不支持酷市场？

技术角度上是酷安做了链接转换，没法成功获取链接。人文角度上是，因为可以直接在设置里修改。

有没有做“去你大爷的XXX”系列的打算？

暂时是应该没有，精力主要放在现在这个App上。这个App完善后，应该也是看看书把Android开发上一些薄弱的地方继续学习下，年前是不会开新坑了。

作为一个Xposed应用，为什么不隐藏应用图标？

本来是有这个打算的，但是考虑到添加规则时可以直接从桌面打开比较方便。
